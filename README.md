# HZZDZZ Couple Finance v1.4.0

## 정리된 구조
- `index.html`: 화면 구조
- `css/style.css`: 화면 스타일
- `app.js`: 앱 시작, 화면 렌더링, 이벤트 연결
- `config.js`: 고정 분류와 기본값
- `utils.js`: 금액·문자열 공통 함수
- `view-period.js`: 각 기기의 연도·월 선택
- `firebase.js`: 공동 Firebase 읽기·저장
- `style.css`: 화면 스타일

## 데이터 호환
- 기존 Firestore 경로 `households/{가계부 ID}` 유지
- 기존 localStorage 키 유지
- 기존 2026년 데이터 및 `yearData` 구조 유지
- 연도·월 화면 선택은 각 기기에만 저장

## GitHub 업로드
저장소 루트에 `index.html`, `README.md`, `css` 폴더, `js` 폴더를 그대로 업로드합니다.
기존 루트의 `app.js`, `firebase.js`, `style.css`는 새 버전이 정상 배포된 뒤 삭제할 수 있습니다.

## v1.4.5 데이터 영구보호
- `firebase-config.js`에 Firebase 설정을 한 번 입력하면 홈 화면 앱 삭제·재설치 후에도 자동 재연결
- 앱 시작 시 Firebase 데이터를 먼저 불러오며, 빈 새 설치 데이터가 기존 공동 데이터를 덮어쓰지 않음
- Firebase 저장 직전 기존 상태를 클라우드 자동 백업으로 보관
- 설정 화면의 `클라우드 이전 백업 복구` 버튼으로 최근 백업 복원 가능

> 중요: `firebase-config.js`의 `DEFAULT_FIREBASE_CONFIG`에 현재 Firebase 설정값을 입력해야 재설치 자동 복구가 동작합니다.

## v1.4.23 공동 동기화 보강
- Firestore `onSnapshot` 실시간 수신 유지
- 저장 중 또는 원격 수신 중 발생한 추가 입력을 대기열에 보관
- 원격 수신 순간에 저장 버튼을 눌러도 변경사항이 누락되지 않도록 수정
- 저장 상태를 `전송 중` → `공동 동기화`로 표시
- 연도·월 및 보기 펼침 상태는 각 휴대폰 로컬에만 유지


## v1.4.24 자투리 통장 누적 방식 변경
- 마감된 월의 `총수입 - 고정지출 - 실제 전체 지출`을 월 잉여금으로 계산
- 월 잉여금이 양수이면 자투리 통장에 적립
- 월 잉여금이 음수이면 자투리 통장에서 차감
- 식비 잔액은 월 잉여금에 이미 포함되므로 별도로 중복 적립하지 않음
- 식비 초과액은 기존처럼 다음 달 식비 적용 예산에서 차감
- 식비의 추가금액과 메모 기능은 기존 방식 그대로 유지

## v1.5.4 월별 CMA 자동 적립 표시
- 화면 명칭을 `관리비 자투리 통장`에서 `자투리 통장`으로 변경했습니다.
- CMA 자동 적립액은 선택한 월까지 마감된 월별 잉여자금만 누적해 표시합니다.
- 달력을 변경하면 해당 월 시점의 CMA 금액이 표시되므로 월별 금액이 서로 구분됩니다.
